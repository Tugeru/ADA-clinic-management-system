# Agents documentation (how to use)

This folder documents **agent roles** used to plan/implement work in the ADA monorepo.

## What to read first

1. `docs/dev-guidelines/Context_Overview.md` — scope and MVP guardrails (single operational role, excluded features).
2. `docs/dev-guidelines/guidelines.md` — definition of done, testing expectations, workflow.
3. Role-specific docs in `docs/agents/*/*.md` — current planning/implementation guidance per role.

## Legacy documents

Files under `docs/agents/**/legacy/` are **archived** and kept for historical reference only. They may not reflect the current repository structure, tooling, or workflow.

If you’re unsure which doc is current, prefer:
- non-legacy files in `docs/agents/*/*.md`
- `docs/dev-guidelines/*`
